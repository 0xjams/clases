## Semana 5
Pensamiento Computacional

{{% note %}}
En esta quinta semana analizaremos el impacto del pensamiento computacional y la capacidad de usar el pensamiento abstracto para simplificar problemas, identificar aspectos relevantes y desarrollar soluciones lógicas de programación que pueden ser ejecutadas por un sistema informático.
{{% /note %}}

---

## ¿Qué es un Problema?
Situación con inconvenientes para llegar a un objetivo.

{{% note %}}
Un problema es una situación que se presenta cuando se busca llegar a un objetivo que ofrece inconvenientes en su camino. Cuando se presenta un problema, se deben usar habilidades cognitivas para buscar estrategias, hacer conjeturas y tomar decisiones.
{{% /note %}}

---

## Elementos Compartidos de Problemas
{{% fragment %}}Estado inicial{{% /fragment %}}
{{% fragment %}}Meta a lograr{{% /fragment %}}
{{% fragment %}}Recursos{{% /fragment %}}
{{% fragment %}}Indicaciones permitidas{{% /fragment %}}
{{% fragment %}}Dominio{{% /fragment %}}
{{% fragment %}}Experiencias similares{{% /fragment %}}

{{% note %}}
La mayoría de los problemas tienen: estado inicial, meta, recursos, indicaciones permitidas, dominio y experiencias similares. Estos elementos son comunes en la mayoría de los problemas que enfrentamos.
{{% /note %}}

---

## Los 4 Pasos de Pólya para la Resolución de Problemas
{{% fragment %}}Entender el problema{{% /fragment %}}
{{% fragment %}}Hacer un plan{{% /fragment %}}
{{% fragment %}}Ejecutar el plan{{% /fragment %}}
{{% fragment %}}Revisar y verificar los resultados{{% /fragment %}}

{{% note %}}
Pasos de Pólya: 1) Entender el problema: reconocer lo preguntado, identificar lo que debe resolverse. 2) Hacer un plan: vincular datos con el problema, determinar recursos, considerar similitudes con otros problemas. 3) Ejecutar el plan: seguir pasos establecidos, replantear si es necesario. 4) Revisar y verificar: comprobar resultados, reflexionar sobre el proceso.
{{% /note %}}

---

## Claves del Pensamiento Computacional
{{% fragment %}}Identificar aspectos abordables con ciencia informática{{% /fragment %}}
{{% fragment %}}Adaptar herramientas tecnológicas{{% /fragment %}}

{{% note %}}
Claves: 1) Identificar qué aspectos del problema se pueden abordar con técnicas informáticas. 2) Adaptar herramientas tecnológicas para usos distintos al original. Estas habilidades son fundamentales en el pensamiento computacional.
{{% /note %}}

---

## Modelo de Inteligencia de Guilford
{{% fragment %}}Pensamiento Convergente{{% /fragment %}}
{{% fragment %}}Pensamiento Divergente{{% /fragment %}}

{{% note %}}
Guilford propone: 1) Pensamiento Convergente: analítico, racional, una solución correcta. Útil para problemas con respuesta precisa. 2) Pensamiento Divergente: abierto, flexible, múltiples soluciones. Esencial para creatividad e innovación. Ambos tipos trabajan juntos en la resolución efectiva de problemas.
{{% /note %}}

---

## Pensamiento Convergente
{{% fragment %}}Resolver problemas de matemáticas{{% /fragment %}}
{{% fragment %}}Establecer presupuestos{{% /fragment %}}
{{% fragment %}}Diseñar estrategias{{% /fragment %}}

{{% note %}}
Ejemplos de pensamiento convergente: resolver problemas matemáticos, seguir recetas, establecer presupuestos, resolver crímenes con lógica, diseñar estrategias de ajedrez. Todos requieren una solución precisa y lógica.
{{% /note %}}

---

## Pensamiento Divergente
{{% fragment %}}Resolver acertijos{{% /fragment %}}
{{% fragment %}}Improvisar en la cocina{{% /fragment %}}
{{% fragment %}}Discutir problemas laborales{{% /fragment %}}

{{% note %}}
Ejemplos de pensamiento divergente: resolver acertijos con juegos de palabras, improvisar en la cocina, discutir problemas laborales y encontrar beneficios inesperados. Estos requieren creatividad y pensamiento fuera de lo común.
{{% /note %}}

---

## Importancia del Pensamiento Computacional
Desglosar problemas complejos en partes manejables.

{{% note %}}
El pensamiento computacional permite entender y resolver problemas eficazmente, mejorando negocios, diseño de productos y soluciones del mundo real. Desarrolla habilidades de pensamiento crítico y analítico.
{{% /note %}}

---

## ¿Qué es el Pensamiento Computacional?
Uso de estrategias de resolución de problemas, algoritmos y modelos.

{{% note %}}
Según Jeannette M. Wing, el pensamiento computacional implica formular problemas y sus soluciones de manera que puedan ser procesadas por un agente de información. Utiliza estrategias de resolución de problemas, algoritmos y modelos.
{{% /note %}}

---

{{% fragment %}}Comprensión de problemas complejos{{% /fragment %}}
{{% fragment %}}Soluciones creativas{{% /fragment %}}
{{% fragment %}}Mejora de habilidades de resolución de problemas{{% /fragment %}}

{{% note %}}
Beneficios: 1) Ayuda a comprender problemas complejos dividiéndolos. 2) Fomenta soluciones creativas. 3) Mejora habilidades generales de resolución de problemas. Estas ventajas se aplican en diversos campos.
{{% /note %}}

{{% fragment %}}Lógica de la programación{{% /fragment %}}

Técnicas principales: lógica de programación y diagramas de flujo. Estas ayudan a estructurar y visualizar los procesos necesarios para resolver problemas de manera eficiente en el contexto computacional.

## ¿Qué es un Algoritmo?
{{% fragment %}}Conjunto de pasos para resolver un problema{{% /fragment %}}

{{% note %}}
Un algoritmo es una secuencia definida de pasos para resolver un problema o realizar una tarea. Es el núcleo de la programación y se puede implementar en diferentes lenguajes.
{{% /note %}}

---

## ¿Qué es un Diagrama de Flujo?
{{% fragment %}}Representación gráfica de un proceso{{% /fragment %}}

{{% note %}}
Un diagrama de flujo es una representación visual de un proceso o algoritmo. Usa símbolos estándar y flechas para mostrar el flujo de acciones y decisiones en un proceso.
{{% /note %}}

---

## Diferencias Clave
{{% fragment %}}Propósito: **Algoritmo** - Resolver problemas. **Diagrama de Flujo** - Visualizar procesos.{{% /fragment %}}
{{% fragment %}}Forma: **Algoritmo** - Texto descriptivo. **Diagrama de Flujo** - Gráfico visual.{{% /fragment %}}
{{% fragment %}}Uso: **Algoritmo** - Implementación en código. **Diagrama de Flujo** - Documentación y planificación.{{% /fragment %}}
{{% fragment %}}Detalles: **Algoritmo** - Instrucciones detalladas. **Diagrama de Flujo** - Visión general del proceso.{{% /fragment %}}

{{% note %}}
Diferencias clave: 1) Propósito: Algoritmos resuelven problemas, diagramas visualizan procesos. 2) Forma: Algoritmos son texto, diagramas son gráficos. 3) Uso: Algoritmos se implementan en código, diagramas documentan y planifican. 4) Detalles: Algoritmos dan instrucciones detalladas, diagramas ofrecen visión general.
{{% /note %}}

---

## Ventajas de los Diagramas de Flujo
{{% fragment %}}Fácil de entender{{% /fragment %}}
{{% fragment %}}Visualiza procesos complejos{{% /fragment %}}
{{% fragment %}}Identifica errores y mejoras{{% /fragment %}}
{{% fragment %}}Facilita la comunicación{{% /fragment %}}

{{% note %}}
Ventajas de diagramas de flujo: 1) Fáciles de entender, incluso para no técnicos. 2) Visualizan procesos complejos claramente. 3) Ayudan a identificar errores y mejoras. 4) Facilitan comunicación entre equipos y partes interesadas.

## Desarrollo del Pensamiento Computacional
Requiere una combinación de conocimientos, habilidades y técnicas.

{{% note %}}
Desarrollar el pensamiento computacional requiere combinar conocimientos, habilidades y técnicas. Implica identificar y comprender el problema, dividirlo en partes, entender los datos necesarios y desarrollar una comprensión de algoritmos y estructuras de datos relevantes.
{{% /note %}}

## Pasos para Resolver un Problema
{{% fragment %}}Desarrollar un algoritmo{{% /fragment %}}
{{% fragment %}}Probar el algoritmo{{% /fragment %}}
{{% fragment %}}Optimización del algoritmo{{% /fragment %}}

{{% note %}}
Pasos: 1) Desarrollar algoritmo: identificar operaciones, diseñar estructura de datos, escribir código. 2) Probar algoritmo: ejecutar en varios escenarios, identificar fallos. 3) Optimizar algoritmo: mejorar rendimiento, velocidad y eficiencia.
{{% /note %}}

## Técnicas Esenciales para Desarrollar el Pensamiento Computacional
{{% fragment %}}Reflexión{{% /fragment %}}
{{% fragment %}}Análisis{{% /fragment %}}
{{% fragment %}}Diseño{{% /fragment %}}
{{% fragment %}}Programación{{% /fragment %}}

{{% note %}}
Técnicas esenciales: 1) Reflexión: evaluar situaciones complejas. 2) Análisis: dividir problemas, identificar patrones. 3) Diseño: desarrollar soluciones creativas. 4) Programación: traducir soluciones a código. 5) Aplicación: adaptar soluciones a nuevos contextos.
{{% /note %}}

## ¿Qué es un Programa?
Conjunto ordenado de instrucciones que realiza una tarea específica.

{{% note %}}
Un programa de computadora es un conjunto de instrucciones escritas en un lenguaje de programación que, al ejecutarse, realiza una tarea específica. La programación es el proceso de crear estas instrucciones.
{{% /note %}}

---

## ¿Qué es Programar?
Proceso de creación de instrucciones informáticas.

{{% note %}}
Programar es crear instrucciones que una computadora puede ejecutar. Implica escribir, compilar y ejecutar código. Es la implementación práctica del pensamiento computacional.
{{% /note %}}

---

## Tipos de Datos y Variables
Diversas formas en que se pueden presentar los valores.

{{% note %}}
Los datos pueden tomar diferentes formas. Las variables son nombres simbólicos que representan valores almacenados. Ejemplos incluyen números, cadenas de texto y listas. Fundamentales en programación.
{{% /note %}}

---

## Sentencias Simples
Instrucciones individuales en un lenguaje de programación.

{{% note %}}
Las sentencias simples son instrucciones individuales en un lenguaje de programación. Ejemplo en Python: `print("Buenos días")`. Son los bloques básicos de construcción de programas.
{{% /note %}}

---

## Diagrama de Flujo
Herramienta para representar procesos

{{% note %}}
Un diagrama de flujo es una herramienta visual para representar pasos de un proceso. Útil para organizar, documentar, tomar decisiones y identificar barreras en un proceso. Esencial en planificación y diseño de algoritmos.
{{% /note %}}

---

## Elementos de un Diagrama de Flujo
{{% fragment %}}Principio y fin{{% /fragment %}}
{{% fragment %}}Flechas o líneas rectas{{% /fragment %}}
{{% fragment %}}Conectores para diagramas extensos{{% /fragment %}}

{{% note %}}
Elementos clave: 1) Principio y fin claramente marcados. 2) Flechas o líneas rectas para indicar dirección. 3) Todas las líneas deben estar conectadas. 4) Usar conectores para diagramas extensos. Estos elementos aseguran claridad y coherencia en el diagrama.
{{% /note %}}
